App delivery

Contexto: La aplicacion de delivery permite a los clientes realizar pedidos de comida desde una variedad de restauran, rastrear el estado del pedido en tiempo real, recibir notificaciones de actualizaciones y comunicarse con el servicio de atencion al cliente si surge algun problema, todo desde la comodidad de su dispositivo movil.

Caso de uso: Realizar un pedido de comida

Actor: Cliente

Precondiciones:

  1. El cliente ha descargado la aplicacion y se ha registrado con exito.
  2. EL cliente tiene acceso a internet.

Flujo principal:

  1. El cliente abre la aplicacion de delivery.
  2. El cliente busca un restaurante o tipo de comida en la barra de busqueda.
  3. La aplicación muestra una lista de restaurantes que coinciden con la búqueda.
  4. El cliente selecciona un restaurante de la lista.
  5. La aplicación muestra el menú del restaurante seleccionado.
  6. El cliente selecciona los platos que desea pedir y los añade a su carrito.
  7. Una vez que el cliente ha terminado de seleccionar platos, revisa su carrito t procede a la pantalla de pago.
  8. El cliente introduce la dirección de entrega y selecciona un método de pago.
  9. El cliente confirma el pedido.
  10. La aplicación muestra una pantalla con confirmació con los detalles del pedido y el tiempo estimado de entrega.
  11. El cliente puede rastrear el estado de su pedido en tiempo real.
  12. El cliente recibe una notificación cuando el pedidos esta cerca de llegar.
  13. El cliente recibe el pedido

Flujo alternativo:

  1. Si el cliente tiene algun problema durante el proceso de pedido, puede contactar con el servicio de atención al cliente a través de la aplicación.
  2. El el cliente quiere cancelar el pedido, puede hacerlo a través de la aplicacion antes de que el restaurante comience a preparar la comida.

Excepciones:

  1. Si el restaurante seleccionado está cerrado o no puede preparar el pedido, la aplicacion notifica al cliente y le pide que seleccione otro restaurante.
  2. Si el método de pago falla, la aplicación pide al cliente que introduzca los detalles de otro metodo de pago.
